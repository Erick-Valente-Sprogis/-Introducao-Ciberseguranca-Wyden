## 3.2.1 O que é Criptografia?

> **Criptografia** é o processo de converter informações em um formato no qual **pessoas não autorizadas não possam lê-las**. Somente uma pessoa confiável, autorizada com a chave secreta ou senha correta, pode descriptografar os dados e acessá-los em sua forma original.

⚠️ Observe que a criptografia **em si não impede** que alguém intercepte os dados — ela só impede que uma pessoa não autorizada **veja ou acesse o conteúdo**.

> 📌 Alguns criminosos podem decidir simplesmente **criptografar seus próprios dados e torná-los inutilizáveis** até que você pague um resgate. _(Isso é exatamente o que o Ransomware faz!)_

---

## 3.2.2 Como você criptografa seus dados?

Programas de software são usados para criptografar arquivos, pastas e até mesmo **unidades inteiras**.

> O **EFS** (_Encrypting File System_ — Sistema de Criptografia de Arquivos) é uma característica do **Windows** que pode criptografar dados. Ele está diretamente vinculado a uma conta de usuário específica — somente o usuário que criptografou os dados poderá acessá-los depois.

### Como usar o EFS no Windows — Passo a Passo

```
Etapa 1 → Selecione um ou mais arquivos ou pastas

Etapa 2 → Clique com o botão direito nos dados selecionados
           e vá para 'Propriedades'

Etapa 3 → Localize e clique em 'Avançado'

Etapa 4 → Marque a caixa 'Criptografar conteúdo
           para proteger os dados'

Etapa 5 → Os arquivos e pastas criptografados com EFS
           são exibidos em verde no Explorer ✅
```

## 3.2.3 Faça Backup de seus Dados

> Ter um backup pode evitar a **perda de dados insubstituíveis**. Para fazer backup corretamente, você precisará de um local de armazenamento adicional e deve copiar os dados para esse local **regularmente**.

### Onde armazenar seu backup?

|Opção|Descrição|Vantagem|
|---|---|---|
|🏠 **Rede residencial**|Armazenar os dados localmente|Controle total sobre os dados|
|💾 **Local secundário**|Dispositivo NAS, HD externo, pen drive, CD, DVD ou fita|Você é o proprietário — responsável pelo custo e manutenção|
|☁️ **A nuvem**|Serviços como Amazon Web Services (AWS)|Dados seguros mesmo em casos de falha de hardware, incêndio ou roubo|

> 💡 Um dos grandes benefícios da nuvem é que seus dados estão protegidos mesmo em **situações extremas** — como incêndio ou roubo do dispositivo físico. O custo varia de acordo com o espaço necessário, exigindo seletividade nos dados de backup.

## 3.2.4 Eles realmente desapareceram?

**Cenário:** Você encontra fotos do usuário anterior no laptop, arrasta para a lixeira, abre a lixeira e clica em "Excluir". As fotos realmente sumiram?

|Opção|Resposta|✅ Correta?|
|:-:|---|:-:|
|A|Sim, as fotos não podem mais ser recuperadas|❌|
|B|Não, as fotos estão apenas inacessíveis no sistema operacional|✅|

> 💡 **Por quê?** Ao "excluir" um arquivo, o sistema operacional apenas **marca aquele espaço como disponível** para ser sobrescrito — os dados continuam fisicamente no disco e podem ser recuperados com ferramentas adequadas até que sejam sobrescritos.

## 3.2.5 Como você exclui os dados permanentemente?

> Você já teve que excluir dados ou se livrar de um disco rígido? Tomou alguma precaução para proteger os dados e evitar que caíssem em mãos erradas?

### 🗑️ Sobrescrita de dados

Para apagar dados de forma que **não sejam mais recuperáveis**, eles devem ser substituídos por **uns e zeros várias vezes**, usando ferramentas específicas:

|Ferramenta|Sistema|
|---|:-:|
|**SDelete**|Windows|
|**Shred**|Linux|
|**Secure Empty Trash**|Mac OS X|

### 💥 Destruição física

> A **única maneira de ter certeza absoluta** de que os dados ou arquivos não são recuperáveis é **destruir fisicamente o disco rígido ou o dispositivo de armazenamento**.

⚠️ Muitos criminosos se valeram de arquivos considerados **impenetráveis ou irrecuperáveis** — nunca subestime a capacidade de recuperação de dados!
