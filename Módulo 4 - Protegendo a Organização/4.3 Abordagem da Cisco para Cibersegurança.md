## 🚨 4.3.1 – O CSIRT da Cisco

> O **CSIRT** (_Computer Security Incident Response Team_) é uma equipe de resposta a incidentes de segurança computacional presente em grandes empresas, responsável pela **recepção, análise e resposta** a incidentes de segurança.

O **Cisco CSIRT** vai além da resposta tradicional, oferecendo:

- Avaliação **proativa** de ameaças
- Planejamento de **mitigação**
- Análise de **tendências de incidentes**
- Revisão da **arquitetura de segurança**

Tudo isso em esforço para **prevenir** a ocorrência de incidentes antes que aconteçam.

## 📒 4.3.2 – Cartilha de segurança

> Uma das melhores maneiras de se preparar para uma violação de segurança é **evitá-la**.

As empresas devem fornecer orientação sobre:

- Como **identificar os riscos** de segurança digital para sistemas, ativos, dados e recursos
- A **implementação de proteções** e treinamento de pessoal
- Um **plano de resposta flexível** que minimiza o impacto e os danos em caso de violação
- As **medidas e processos** de segurança que precisam ser implementados após uma violação

> Todas estas informações devem ser compiladas em um **manual de segurança**.

---

### 📖 Manual de segurança

> Um manual de segurança é um conjunto de **consultas ou relatórios** que descrevem um processo padronizado de detecção e resposta a incidentes.

Idealmente, um manual de segurança deve:

- **Destacar** como identificar e automatizar a resposta a ameaças comuns, como a detecção de máquinas infectadas por malware, atividade de rede suspeita ou tentativas de autenticação irregular
- **Descrever e definir** claramente o tráfego de entrada e saída
- **Fornecer informações resumidas**, incluindo tendências, estatísticas e contagens
- **Fornecer acesso utilizável e rápido** às principais estatísticas e métricas
- **Correlacionar eventos** em todas as fontes de dados relevantes

## 🔧 4.3.3 – Ferramentas para prevenção e detecção de incidentes

> Estas são algumas das ferramentas usadas para detectar e prevenir incidentes de segurança:

---

### 📊 SIEM — Security Information and Event Management

> Um sistema **SIEM** coleta e analisa **alertas de segurança, logs** e outros dados históricos e em tempo real de dispositivos de segurança na rede para facilitar a **detecção precoce de ataques cibernéticos**.

---

### 🔒 DLP — Data Loss Prevention

> Um sistema **DLP** é projetado para **impedir que dados confidenciais sejam roubados ou escapem** de uma rede.

Ele monitora e protege os dados em três estados:

|Estado|Descrição|
|---|---|
|**Dados em uso**|Dados sendo acessados por um usuário|
|**Dados em movimento**|Dados que viajam pela rede|
|**Dados inativos**|Dados armazenados em uma rede ou dispositivo|

## 🔐 4.3.4 – Cisco ISE e TrustSec

> O **Cisco Identity Services Engine (ISE)** e o **TrustSec** reforçam o acesso do usuário aos recursos de rede ao criar **políticas de controle de acesso por função**.

---

### 🎬 Cisco Network Visibility and Segmentation (NVS)

> À medida que as redes crescem — com mais usuários, dispositivos, acesso móvel e nuvem — torna-se mais difícil detectar quando houve uma violação. O **NVS** oferece um novo nível de visibilidade para identificar e conter ameaças em tempo real.

O NVS é composto por três soluções que trabalham em conjunto:

```
┌─────────────────────────────────────────────────────────────────┐
│                    Cisco NVS                                     │
│                                                                  │
│  ┌─────────────────┐  ┌──────────────────┐  ┌───────────────┐  │
│  │  StealthWatch   │  │       ISE        │  │   TrustSec    │  │
│  │                 │  │                  │  │               │  │
│  │ Usa Machine     │  │ Visibilidade de  │  │ Simplifica    │  │
│  │ Learning para   │  │ usuários e       │  │ controle de   │  │
│  │ analisar tráfego│  │ dispositivos;    │  │ acesso;       │  │
│  │ — detecta       │  │ gerenciador de   │  │ aplica        │  │
│  │ ameaças mesmo   │  │ políticas        │  │ segmentação   │  │
│  │ criptografadas  │  │ central          │  │ automática e  │  │
│  │                 │  │                  │  │ quarentena    │  │
│  └─────────────────┘  └──────────────────┘  └───────────────┘  │
│                                                                  │
│         Visibilidade + Controle + Automação em tempo real       │
└─────────────────────────────────────────────────────────────────┘
```

|Solução|Função principal|
|---|---|
|**StealthWatch**|Usa Machine Learning para analisar tráfego e detectar ameaças, mesmo quando criptografadas|
|**ISE**|Aprofunda a visibilidade sobre usuários e dispositivos; atua como gerenciador de políticas central|
|**TrustSec**|Trabalha com o ISE para simplificar o controle de acesso e aplicar automaticamente a política de segmentação em toda a rede para quarentena de ameaças|

## 🧠 4.3.5 – Conheça a linguagem (atividade)

> Este módulo contém muitas informações técnicas e jargões essenciais para a carreira em segurança digital. Associe cada descrição à tecnologia correta.

|Descrição|Resposta|
|---|:-:|
|Bloqueia ou nega o tráfego com base em uma correspondência de regra ou assinatura positiva|**IPS**|
|Um sistema projetado para impedir que dados confidenciais sejam roubados ou escapem de uma rede|**DLP**|
|Um sistema que coleta e analisa alertas de segurança, registros e outros dados históricos e em tempo real de dispositivos de segurança na rede|**SIEM**|
|Verifica os dados em um banco de dados de regras ou assinaturas de ataque, registra quaisquer detecções e cria um alerta para o administrador da rede|**IDS**|
